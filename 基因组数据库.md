# 一、docker

## 1.1安装

``` bash
sudo apt update #更新源 
sudo apt install apt-transport-https ca-certificates curl gnupg-agent software-properties-common 
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo apt-key add – #报错 
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /usr/share/keyrings/docker-archive-keyring.gpg 
sudo add-apt-repository "deb [arch=amd64] https://download.docker.com/linux/ubuntu $(lsb_release -cs) stable" #报错

sudo apt-key adv --keyserver keyserver.ubuntu.com --recv-keys 7EA0A9C3F273FCD8
sudo apt install docker-ce docker-ce-cli containerd.io 
sudo usermod -aG docker username
```

第一种方法失败

### 若不可，则使用以下代码

``` bash
# 更新apt包索引并安装一些允许apt通过HTTPS使用软件库的包
sudo apt-get update
sudo apt-get install \
    ca-certificates \
    curl \
    gnupg \
    lsb-release
    
# 添加Docker的官方GPG密钥
sudo mkdir -p /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg

# 设置稳定的软件库
echo \
  "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/ubuntu \
  $(lsb_release -cs) stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null
  
# 更新apt包索引
sudo apt-get update

# 安装Docker引擎、Docker CLI和containerd
sudo apt-get install docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin

sudo usermod -aG docker username
```

在某些系统中，默认使用的是nf_tables（新的iptables实现），这可能导致与Docker的兼容性问题。可以尝试切换到传统的iptables实现：

``` bash
sudo update-alternatives --set iptables /usr/sbin/iptables-legacy
sudo update-alternatives --set ip6tables /usr/sbin/ip6tables-legacy
```

``` bash
git clone http://git.genek.cn:3333/zhxd2/GenomeDB.git
```

## 2.2构建镜像

### 2.2.1方法一 从Dockerfile构建

``` bash

```

## 2.3启动容器

